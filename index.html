Informasi & Kompleksitas Algoritma

O(n)

Linear Search

Mencari data mahasiswa berdasarkan Nama. Algoritma memindai setiap baris data dari awal hingga akhir secara sekuensial. Kecepatannya berbanding lurus dengan jumlah data ($n$).

O(log n)

Binary Search

Mencari mahasiswa berdasarkan NIM. Data diurutkan berdasarkan NIM terlebih dahulu, kemudian pencarian dilakukan dengan membagi wilayah pencarian menjadi dua bagian secara berulang.

O(n)

Sequential Search

Mencari mahasiswa berdasarkan Program Studi. Secara teknis sama dengan Linear Search, algoritma akan membandingkan setiap data secara berurutan.

O(n²)

Bubble Sort

Mengurutkan berdasarkan IPK. Algoritma berulang kali membandingkan elemen bersebelahan dan menukarnya jika posisi salah.

O(n²)

Selection Sort

Mengurutkan berdasarkan IPK. Membagi data menjadi bagian terurut dan tidak terurut, lalu mencari elemen minimum/maksimum berulang kali.

O(n log n)

Merge Sort

Mengurutkan berdasarkan IPK. Algoritma Divide and Conquer yang membagi daftar menjadi bagian-bagian kecil lalu menggabungkannya secara terurut.
