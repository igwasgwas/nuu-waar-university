Informasi & Kompleksitas Algoritma

O(n)

Linear Search

Mencari data mahasiswa berdasarkan Nama. Algoritma memindai setiap baris data dari awal hingga akhir secara sekuensial. Kecepatannya berbanding lurus dengan jumlah data ($n$).

O(log n)

Binary Search

Mencari mahasiswa berdasarkan NIM. Data diurutkan berdasarkan NIM terlebih dahulu, kemudian pencarian dilakukan dengan membagi wilayah pencarian menjadi dua bagian secara berulang. Sangat efisien untuk data besar.

O(n²)

Bubble Sort

Mengurutkan peringkat IPK mahasiswa. Algoritma berulang kali membandingkan elemen yang bersebelahan dan menukarnya jika posisinya salah, terus berulang hingga semua data terurut rapi.
